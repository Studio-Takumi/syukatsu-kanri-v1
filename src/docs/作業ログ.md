### ステータス管理の論理設計の変更 🔧

#### 問題点

- 論理的に同時に存在し得ないステータス（未応募と応募済み）が別カテゴリになっていた
- DBのレコード数が無駄に増える

#### 解決策

- **ステータスカテゴリを論理的なものに整理**（6種類に削減）
    - `apply` - 未応募/応募済み（is_checkedで判定）
    - `submission` - 要ES・要テスト/ES提出済・テスト提出済（is_checkedで判定）
    - `event` - 説明会・面接予定
    - `selection` - 選考待ち/選考通過（is_checkedで判定）
    - `passed` - 合格
    - `rejected` - 選考落ち

- **is_checkedフラグで完了/未完了を管理**
    - `is_checked: false` → 濃い色（未完了状態）
    - `is_checked: true` → 薄い色（完了状態）

- **StatusBadgeコンポーネントで描画時に判定**
    - `STATUS_COLORS`: `unchecked` / `checked` で色を分ける
    - `STATUS_LABELS`: `unchecked` / `checked` で表示名を分ける
    - コンポーネント内で `status.is_checked` を見て自動切り替え
